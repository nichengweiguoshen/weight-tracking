# 体重追踪

一个移动端优先的本地体重记录工具，用体重、腰围、训练/饮食执行情况和 EMA 趋势线帮助判断近期体重变化。

## 当前功能

- 记录任意日期的体重、腰围、训练执行、饮食执行和复盘备注。
- 支持 kg / 斤切换，数据统一按 kg 存储。
- 支持设置目标体重和目标日期，并计算每周建议变化速度。
- 支持补录：在记录页选择日期后保存即可。
- 支持编辑：在趋势页的历史记录中点击记录或编辑按钮，会回到记录页修改。
- 支持删除：历史记录中的删除按钮会删除对应日期的数据。
- 支持 JSON / CSV 导出，以及 JSON / CSV 导入恢复。
- 趋势页展示 EMA7、EMA21、累计变化、BMI 变化、目标进度、腰围趋势、连续打卡和近 7 日缺失天数。
- 趋势诊断会结合目标进度，显示剩余体重和建议速度。
- 支持记录训练分钟、热量偏差、睡眠时长和波动原因标签。
- 支持近 7 日周报、平台期识别、过快下降提醒和异常波动解释。

## 使用方式

直接用浏览器打开 `index.html` 即可使用。也可以用任意静态服务器托管这个文件。

```powershell
python -m http.server 8080
```

然后访问 `http://localhost:8080`。

## 数据说明

数据保存在浏览器本地 IndexedDB，数据库名为 `WeightTrackerDB`。身高和单位偏好保存在 `localStorage`。

注意：

- 清理浏览器站点数据可能会删除记录。
- 换浏览器或换设备不会自动同步数据。
- 建议定期使用导出按钮备份 JSON 或 CSV。

## 导入导出

JSON 导出包含记录、身高、单位和目标设置，适合完整备份。

CSV 导出只包含记录明细，字段为：

```csv
date,weight_kg,waist_cm,workout,diet,workout_minutes,calorie_delta,sleep_hours,factors,note,timestamp
```

导入时会按日期覆盖同一天的旧记录。JSON 导入如果包含目标和身高，也会同步恢复这些设置。

## 分析说明

- 平台期：最近两周均值变化较小，同时执行分不低时触发。
- 过快下降：最近一周下降速度超过常见安全区间时触发。
- 异常波动：单日体重跳变较大时触发，优先结合高盐、高碳、睡眠差、训练恢复等标签解释。
- 周报：基于最近 7 条记录统计均重变化、腰围变化、训练总量、平均睡眠和高频标签。

## 依赖

页面通过 CDN 加载以下库：

- Tailwind CSS
- Chart.js
- Dexie.js
- Phosphor Icons

因此首次加载需要网络连接；加载完成后的记录数据仍保存在本地。
